# Plan for Firebase Deploy

## Goal
Deploy the StreamParty React app to Firebase Hosting.

## Prerequisites
- Firebase CLI installed globally (`npm install -g firebase-tools`)
- Firebase account login
- Firebase project initialized in the directory
- Build the app successfully

## Steps

### 1. Install Firebase CLI (if not installed)
Run `npm install -g firebase-tools` to install the Firebase CLI globally.

### 2. Login to Firebase
Run `firebase login` to authenticate with your Firebase account.

### 3. Initialize Firebase in the project
Run `firebase init` in the project directory.
- Select "Hosting" as the feature to set up.
- Choose the existing Firebase project "casebuddy-transcribe-135-5c092".
- Set the public directory to `dist` (the build output).
- Configure as a single-page app (SPA) with `index.html` as the entry point.
- Do not overwrite `index.html` when prompted.

This will create `firebase.json` and `.firebaserc` files.

### 4. Build the app
Run `npm run build` to create the production build in the `dist` directory.

### 5. Deploy to Firebase Hosting
Run `firebase deploy` to deploy the app to Firebase Hosting.
- This will upload the contents of `dist` to Firebase Hosting.
- After deployment, Firebase will provide the hosting URL.

## Post-Deployment
- Verify the app loads correctly at the provided URL.
- Test login functionality, video sync, and VK video support.
- If issues arise (e.g., auth domain), update Firebase auth settings to allow the hosting domain.

## Potential Issues
- Auth domain: The app's authDomain is set to "casebuddy-transcribe-135-5c092.firebaseapp.com". If deploying to a custom domain, update Firebase auth settings.
- Build errors: Ensure all dependencies are installed and the build completes without errors.
- CORS or other issues: May need to configure Firebase rules or settings.

## Notes
- The app uses Firebase for auth and Firestore, which are already configured.
- VK token is stored in env; ensure VITE_VK_TOKEN is set in production if needed.
